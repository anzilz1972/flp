## 教程

作为本教程的激励示例，你将编写一个文字冒险游戏。这是一种交互式小说，用户输入文本命令与虚构世界互动，并接收关于发生事件的文本描述。命令将简化版的自然语言，如 `get sword`、`attack dragon`、`go north`、`enter shop` 或 `buy cheese`。

### 匹配序列

你的主循环需要从用户获取输入并将其拆分为单词，假设是一个字符串列表，如下所示：

```python
command = input("What are you doing next? ")
# 分析 command.split() 的结果
```

下一步是解释这些单词。大多数命令都有两个单词：一个动作和一个对象。所以你可能会尝试这样做：

```python
[action, obj] = command.split()
... # 解释 action, obj
```

这行代码的问题是缺少了一些东西：如果用户输入多于或少于2个单词怎么办？为了防止这个问题，你可以检查单词列表的长度，或者捕获上述语句会引发的 `ValueError`。

你可以用 match 语句代替：

```python
match command.split():
    case [action, obj]:
        ... # 解释 action, obj
```

match 语句评估**"主题"**（`match` 关键字后面的值），并根据**模式**（`case` 旁边的代码）进行检查。模式能够做两件不同的事情：

- 验证主题具有某种结构。在你的例子中，`[action, obj]` 模式匹配任意恰好两个元素的序列。这叫做**匹配**
- 它将模式中的一些名称绑定到主题的组成元素。在这种情况下，如果列表有两个元素，它将绑定 `action = subject[0]` 和 `obj = subject[1]`。

如果有匹配，case 块内的语句将在绑定变量的前提下执行。如果没有匹配，则继续执行 match 之后的语句。

注意，与解包赋值类似，你可以使用括号、方括号或逗号分隔作为同义词。所以你可以写 `case action, obj` 或 `case (action, obj)`，含义相同。所有形式都将匹配任意序列（例如列表或元组）。

### 匹配多个模式

即使大多数命令采用动作/对象形式，你可能也想添加不同长度的用户命令。例如，你可能想添加不带对象的单独动词，如 `look` 或 `quit`。match 语句可以（也很可能）有多个 `case`：

```python
match command.split():
    case [action]:
        ... # 解释单独动词动作
    case [action, obj]:
        ... # 解释动作和对象
```

match 语句从上到下检查模式。如果模式与主题不匹配，则尝试下一个模式。但是，一旦找到*第一个*匹配的模式，就会执行该 case 的主体，并忽略所有后续 case。这类似于 `if/elif/elif/...` 语句的工作方式。

### 匹配特定值

你的代码仍然需要查看特定的动作，并根据具体动作（例如 `quit`、`attack` 或 `buy`）有条件地执行不同的逻辑。你可以使用 `if/elif/elif/...` 链或函数字典来实现，但在这里我们将利用模式匹配来解决这个任务。你可以在模式中使用字面量值（如 `"quit"`、`42` 或 `None`）而不是变量。这样你就可以写：

```python
match command.split():
    case ["quit"]:
        print("Goodbye!")
        quit_game()
    case ["look"]:
        current_room.describe()
    case ["get", obj]:
        character.get(obj, current_room)
    case ["go", direction]:
        current_room = current_room.neighbor(direction)
    # 其余命令放在这里
```

像 `["get", obj]` 这样的模式只匹配第一个元素等于 `"get"` 的两元素序列。它还会绑定 `obj = subject[1]`。

正如你在 `go` case 中看到的，我们也可以在不同的模式中使用不同的变量名。

字面量值使用 `==` 运算符进行比较，但常量 `True`、`False` 和 `None` 使用 `is` 运算符进行比较。

### 匹配多个值

玩家可能能够通过一系列命令 `drop key`、`drop sword`、`drop cheese` 来丢弃多个物品。这种界面可能很麻烦，你可能希望允许在单个命令中丢弃多个物品，如 `drop key sword cheese`。在这种情况下，你事先不知道命令中会有多少个单词，但你可以在模式中使用扩展解包，就像在赋值中允许的那样：

```python
match command.split():
    case ["drop", *objects]:
        for obj in objects:
            character.drop(obj, current_room)
    # 其余命令放在这里
```

这将匹配以 "drop" 作为第一个元素的任何序列。所有剩余的元素将被捕获到一个 `list` 对象中，绑定到 `objects` 变量。

此语法与序列解包有类似的限制：你不能在模式中有多个带星号的名称。

### 添加通配符

你可能想在所有模式都失败时打印一条错误消息，说明命令无法识别。你可以使用我们刚学到的功能，写 `case [*ignored_words]` 作为最后一个模式。但是有一个更简单的方法：

```python
match command.split():
    case ["quit"]: ... # 为简洁省略代码
    case ["go", direction]: ...
    case ["drop", *objects]: ...
    ... # 其他 case
    case _:
        print(f"Sorry, I couldn't understand {command!r}")
```

这个特殊模式写成 `_`（称为通配符）总是匹配的，但不会绑定任何变量。

注意，它可以匹配任何对象，而不仅仅是序列。因此，把它单独放在最后一个模式中才有意义（为了防止错误，Python 会阻止你在它之前使用它）。

### 组合模式

现在是退后一步，理解你一直在使用的模式是如何构建的好时机。模式可以相互嵌套，我们在上面的例子中隐式地做到了这一点。

有一些"简单"模式（这里的"简单"意味着它们不包含其他模式）：

- **捕获模式**（像 `direction`、`action`、`objects` 这样的独立名称）。我们没有单独讨论过这些，而是将它们作为其他模式的一部分使用。
- **字面量模式**（字符串字面量、数字字面量、`True`、`False` 和 `None`）
- **通配符模式** `_`

到目前为止，我们唯一试验过的非简单模式是序列模式。序列模式中的每个元素实际上可以是任何其他模式。这意味着你可以写一个像 `["first", (left, right), _, *rest]` 这样的模式。这将匹配至少三个元素的主题，其中第一个元素等于 `"first"`，第二个元素本身是一个两个元素的序列。它还会绑定 `left=subject[1][0]`、`right=subject[1][1]` 和 `rest = subject[3:]`。

### 或模式

回到冒险游戏的例子，你可能希望有几个模式产生相同的结果。例如，你可能希望命令 `north` 和 `go north` 等效。你也可能希望为 `get X`、`pick up X` 和 `pick X up` 提供别名。

模式中的 `|` 符号将它们组合为替代项。例如，你可以写：

```python
match command.split():
    ... # 其他 case
    case ["north"] | ["go", "north"]:
        current_room = current_room.neighbor("north")
    case ["get", obj] | ["pick", "up", obj] | ["pick", obj, "up"]:
        ... # 拾取给定对象的代码
```

这称为**或模式**，将产生预期的结果。模式从左到右尝试；如果有多个替代项匹配，这可能与知道绑定什么有关。编写或模式时的一个重要限制是，所有替代项应该绑定相同的变量。所以 `[1, x] | [2, y]` 是不允许的，因为这会使匹配成功后绑定哪个变量变得模糊。`[1, x] | [2, x]` 是完全允许的，如果成功将始终绑定 `x`。

### 捕获匹配的子模式

我们 "go" 命令的第一个版本是用 `["go", direction]` 模式编写的。我们在最后一个版本中使用模式 `["north"] | ["go", "north"]` 所做的更改有一些好处，但与原版相比也有一些缺点：最新版本允许别名，但方向是硬编码的，这将迫使我们为 north/south/east/west 实际使用单独的模式。这会导致一些代码重复，但同时我们获得了更好的输入验证，而且如果用户输入的是 `"go figure!"` 而不是方向，我们就不会进入该分支。

我们可以尝试两全其美，方法是（为简洁起见，我省略没有 "go" 的别名版本）：

```python
match command.split():
    case ["go", ("north" | "south" | "east" | "west")]:
        current_room = current_room.neighbor(...)
        # 我怎么知道往哪个方向走？
```

这段代码是单个分支，它验证 "go" 后面的单词确实是一个方向。但是移动玩家的代码需要知道选择了哪一个，目前没有办法做到。我们需要的是一个行为类似于或模式但同时进行捕获的模式。我们可以用**as 模式**做到这一点：

```python
match command.split():
    case ["go", ("north" | "south" | "east" | "west") as direction]:
        current_room = current_room.neighbor(direction)
```

as 模式匹配其左侧的任何模式，但也将值绑定到一个名称。

### 向模式添加条件

我们上面探索的模式可以做一些强大的数据过滤，但有时你可能希望获得布尔表达式的全部能力。假设你实际上希望只在基于 `current_room` 的可能出口的受限方向集合中允许 "go" 命令。我们可以通过在 case 中添加**守卫**来实现。守卫由 `if` 关键字后跟任意表达式组成：

```python
match command.split():
    case ["go", direction] if direction in current_room.exits:
        current_room = current_room.neighbor(direction)
    case ["go", _]:
        print("Sorry, you can't go that way")
```

守卫不是模式的一部分，而是 case 的一部分。它只在模式匹配后检查，并且在所有模式变量都被绑定之后（这就是为什么在上面的例子中条件可以使用 `direction` 变量）。如果模式匹配且条件为真，则正常执行 case 的主体。如果模式匹配但条件为假，则 match 语句继续检查下一个 case，就好像模式没有匹配一样（可能有已绑定一些变量的副作用）。

### 添加UI：匹配对象

你的冒险游戏大获成功，你被要求实现一个图形界面。你选择的UI工具包允许你编写一个事件循环，通过调用 `event.get()` 获取新的事件对象。生成的对象可以根据用户操作具有不同的类型和属性，例如：

- 当用户按键时生成 `KeyPress` 对象。它有一个 `key_name` 属性，包含所按按键的名称，以及一些关于修饰键的其他属性。
- 当用户点击鼠标时生成 `Click` 对象。它有一个 `position` 属性，包含指针的坐标。
- 当用户点击游戏窗口的关闭按钮时生成 `Quit` 对象。

你可以使用模式来识别不同类型的对象，而不是编写多个 `isinstance()` 检查，并且也可以对其属性应用模式：

```python
match event.get():
    case Click(position=(x, y)):
        handle_click_at(x, y)
    case KeyPress(key_name="Q") | Quit():
        game.quit()
    case KeyPress(key_name="up arrow"):
        game.go_north()
    ...
    case KeyPress():
        pass # 忽略其他按键
    case other_event:
        raise ValueError(f"Unrecognized event: {other_event}")
```

像 `Click(position=(x, y))` 这样的模式只匹配事件的类型是 `Click` 类的子类的情况。它还要求事件有一个与 `(x, y)` 模式匹配的 `position` 属性。如果有匹配，局部变量 `x` 和 `y` 将获得期望的值。

像 `KeyPress()` 这样没有参数的模式将匹配 `KeyPress` 类的任何实例。模式中只指定你想要的属性进行匹配，任何其他属性都会被忽略。

### 匹配位置属性

上一节描述了在进行对象匹配时如何匹配命名属性。对于某些对象，描述匹配的参数可能按位置会很方便（特别是如果只有少数属性并且它们有一个"标准"顺序的话）。如果你使用的类是命名元组或数据类，你可以按照构造对象时使用的相同顺序来做到这一点。例如，如果上面的UI框架这样定义它们的类：

```python
from dataclasses import dataclass

@dataclass
class Click:
    position: tuple
    button: Button
```

那么你可以将上面的 match 语句重写为：

```python
match event.get():
    case Click((x, y)):
        handle_click_at(x, y)
```

`(x, y)` 模式将自动与 `position` 属性匹配，因为模式中的第一个参数对应于数据类定义中的第一个属性。

其他类没有属性的自然顺序，所以你需要在模式中使用显式名称来与它们的属性匹配。但是，可以手动指定允许位置匹配的属性的顺序，就像这个替代定义一样：

```python
class Click:
    __match_args__ = ("position", "button")
    def __init__(self, pos, btn):
        self.position = pos
        self.button = btn
        ...
```

`__match_args__` 特殊属性定义了属性的显式顺序，可以在模式中使用，如 `case Click((x,y))`。

### 匹配常量和枚举

上面的模式将所有鼠标按钮视为相同的，你决定要接受左键点击，而忽略其他按钮。在这样做的过程中，你注意到 `button` 属性被类型化为 `Button`，这是一个用 `enum.Enum` 构建的枚举。你实际上可以像这样匹配枚举值：

```python
match event.get():
    case Click((x, y), button=Button.LEFT):  # 这是一个左键点击
        handle_click_at(x, y)
    case Click():
        pass  # 忽略其他点击
```

这适用于任何带点号的名称（如 `math.pi`）。但是，非限定名称（即没有点的裸名称）将始终被解释为捕获模式，因此为避免这种歧义，请在模式中始终使用带限定符的常量。

### 走向云端：映射

你决定制作一个在线版本的游戏。所有逻辑都在服务器上，UI 在客户端，客户端将通过 JSON 消息进行通信。通过 `json` 模块，这些将被映射到 Python 字典、列表和其他内置对象。

我们的客户端将接收一个字典列表（从 JSON 解析），其中包含要采取的操作，每个元素看起来像这样：

- `{"text": "The shop keeper says 'Ah! We have Camembert, yes sir'", "color": "blue"}`
- 如果客户端应该暂停 `{"sleep": 3}`
- 播放声音 `{"sound": "filename.ogg", "format": "ogg"}`

到目前为止，我们的模式都是处理序列的，但也有基于其存在的键来匹配映射的模式。在这种情况下，你可以使用：

```python
for action in actions:
    match action:
        case {"text": message, "color": c}:
            ui.set_text_color(c)
            ui.display(message)
        case {"sleep": duration}:
            ui.wait(duration)
        case {"sound": url, "format": "ogg"}:
            ui.play(url)
        case {"sound": _, "format": _}:
            warning("Unsupported audio format")
```

映射模式中的键必须是字面量，但值可以是任何模式。与序列模式一样，所有子模式都必须匹配，总体模式才能匹配。

你可以在映射模式中使用 `**rest` 来捕获主题中的额外键。请注意，如果你省略了这个，匹配时将忽略主题中的额外键，即消息 `{"text": "foo", "color": "red", "style": "bold"}` 将匹配上面示例中的第一个模式。

### 匹配内置类

上面的代码可以使用一些验证。由于消息来自外部源，字段的类型可能是错误的，导致错误或安全问题。

任何类都是有效的匹配目标，包括内置类如 `bool`、`str` 或 `int`。这允许我们将上面的代码与类模式结合起来。所以我们可以用 `{"text": str() as message, "color": str() as c}` 而不是写 `{"text": message, "color": c}` 来确保 `message` 和 `c` 都是字符串。对于许多内置类（完整列表见 [PEP 634](../pep-0634/)），你可以使用位置参数作为简写，写 `str(c)` 而不是 `str() as c`。完整重写的版本如下：

```python
for action in actions:
    match action:
        case {"text": str(message), "color": str(c)}:
            ui.set_text_color(c)
            ui.display(message)
        case {"sleep": float(duration)}:
            ui.wait(duration)
        case {"sound": str(url), "format": "ogg"}:
            ui.play(url)
        case {"sound": _, "format": _}:
            warning("Unsupported audio format")
```

## 附录 A – 快速入门

match 语句接受一个表达式，并将其值与作为 case 块给出的后续模式进行比较。这表面上看类似于 C、Java 或 JavaScript（以及许多其他语言）中的 switch 语句，但功能强大得多。

最简单的形式是将主题值与一个或多个字面量进行比较：

```python
def http_error(status):
    match status:
        case 400:
            return "Bad request"
        case 404:
            return "Not found"
        case 418:
            return "I'm a teapot"
        case _:
            return "Something's wrong with the Internet"
```

注意最后一个块：*"变量名"* `_` 充当**通配符**，永远不会匹配失败。

你可以使用 `|`（"或"）在单个模式中组合多个字面量：

```python
case 401 | 403 | 404:
    return "Not allowed"
```

模式看起来像解包赋值，可用于绑定变量：

```python
# point 是一个 (x, y) 元组
match point:
    case (0, 0):
        print("Origin")
    case (0, y):
        print(f"Y={y}")
    case (x, 0):
        print(f"X={x}")
    case (x, y):
        print(f"X={x}, Y={y}")
    case _:
        raise ValueError("Not a point")
```

仔细研究这个例子！第一个模式有两个字面量，可以被认为是上面所示的字面量模式的扩展。但是接下来的两个模式组合了一个字面量和一个变量，变量**绑定**了主题（`point`）中的一个值。第四个模式捕获两个值，这使其在概念上类似于解包赋值 `(x, y) = point`。

如果你使用类来构建数据结构，你可以使用类名后跟一个参数列表，类似于构造函数，但有能力将属性捕获到变量中：

```python
from dataclasses import dataclass

@dataclass
class Point:
    x: int
    y: int

def where_is(point):
    match point:
        case Point(x=0, y=0):
            print("Origin")
        case Point(x=0, y=y):
            print(f"Y={y}")
        case Point(x=x, y=0):
            print(f"X={x}")
        case Point():
            print("Somewhere else")
        case _:
            print("Not a point")
```

你可以对某些为其属性提供顺序的内置类（如数据类）使用位置参数。你也可以通过在你的类中设置 `__match_args__` 特殊属性来为模式中的属性定义特定顺序。如果它设置为 ("x", "y")，以下模式都是等价的（并且都绑定 `y` 属性到 `var` 变量）：

```python
Point(1, var)
Point(1, y=var)
Point(x=1, y=var)
Point(y=var, x=1)
```

模式可以任意嵌套。例如，如果我们有一个短点列表，我们可以这样匹配：

```python
match points:
    case []:
        print("No points")
    case [Point(0, 0)]:
        print("The origin")
    case [Point(x, y)]:
        print(f"Single point {x}, {y}")
    case [Point(0, y1), Point(0, y2)]:
        print(f"Two on the Y axis at {y1}, {y2}")
    case _:
        print("Something else")
```

我们可以在模式中添加一个 `if` 子句，称为"守卫"。如果守卫为假，`match` 会继续尝试下一个 case 块。注意，值捕获发生在守卫评估之前：

```python
match point:
    case Point(x, y) if x == y:
        print(f"Y=X at {x}")
    case Point(x, y):
        print(f"Not on the diagonal")
```

其他关键特性：

- 与解包赋值一样，元组和列表模式具有完全相同的含义，实际上匹配任意序列。一个重要的例外是它们不匹配迭代器或字符串。（从技术上讲，主题必须是 `collections.abc.Sequence` 的实例。）
- 序列模式支持通配符：`[x, y, *rest]` 和 `(x, y, *rest)` 的工作方式类似于解包赋值中的通配符。`*` 后的名称也可以是 `_`，因此 `(x, y, *_)` 匹配至少两个项目而不绑定剩余项目。
- 映射模式：`{"bandwidth": b, "latency": l}` 从字典中捕获 `"bandwidth"` 和 `"latency"` 的值。与序列模式不同，额外的键会被忽略。也支持通配符 `**rest`。（但 `**_` 会是冗余的，所以不允许。）
- 子模式可以使用 `as` 关键字捕获：
  ```python
  case (Point(x1, y1), Point(x2, y2) as p2): ...
  ```
- 大多数字面量通过相等性比较，然而单例 `True`、`False` 和 `None` 通过身份比较。
- 模式可以使用命名常量。这些必须是用点号分隔的名称，以防止被解释为捕获变量：
  ```python
  from enum import Enum
  class Color(Enum):
      RED = 0
      GREEN = 1
      BLUE = 2
  
  match color:
      case Color.RED:
          print("I see red!")
      case Color.GREEN:
          print("Grass is green")
      case Color.BLUE:
          print("I'm feeling the blues :(")
  ```
